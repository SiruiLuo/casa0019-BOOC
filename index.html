BoOC

A library occupancy monitoring system

DATASET
We use data from UCL API 1.4.11, which includes the following features: A real-time summary (with one-minute accuracy) of the number of seats available and occupied in each library section for students. A detailed list of sensors in every library map, providing the average utilization of the library in 10-minute intervals throughout the previous week.

DATA DEVICE (Physical)
 Our physical device allows users to switch between three different libraries using three buttons. A pointer dynamically displays the real-time occupancy rate of seats in the selected library, helping users understand the current library usage. An e-ink screen provides additional information, including the library name, floor plan, the number of free and occupied seats, and the overall occupancy rate. This design ensures users can quickly grasp both the library layout and its usage status, all in real-time.

DATA DEVICE (Digital)
 Enhanced Dashboard: The digital dashboard includes all features from the physical device while offering additional functionalities: Real-Time Visualizations: The dashboard displays text summaries and pie charts to illustrate the current seat occupancy rate. Historical Analysis: A line chart shows the hourly average occupancy rates over the past week, offering an in-depth understanding of library usage patterns over time. Flexible Dimensions: Users can analyze library data from multiple perspectives, enabling better decision-making for long-term planning.

HOW TO
 Open the application and scan the trigger with your smartphone camera. Use the buttons on the device to switch between different libraries. The dashboard will automatically update to reflect data from the selected library, ensuring a seamless user experience.
